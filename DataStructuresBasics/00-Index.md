# Data Structures Basics Drill Folder

This folder is a beginner-first practice section for common interview basics that are easy to miss under pressure.

## Why this folder exists
- Sometimes interviewers do not ask deep theory first.
- They ask simple things like: create a `HashMap`, iterate it, print keys and values, traverse an array, or explain tree traversals.
- This folder is meant to make those basics automatic.

## Files in this folder
1. `HashMapBasicsDemo.java`
2. `LinkedListBasicsDemo.java`
3. `ArraysBasicsDemo.java`
4. `BinaryTreeBasicsDemo.java`

## What each file covers
- `HashMapBasicsDemo.java`: create, add, update, get, remove, print, and iterate a `HashMap` in multiple ways.
- `LinkedListBasicsDemo.java`: create, add, remove, print, iterate, and use `LinkedList` as a list, queue, and stack.
- `ArraysBasicsDemo.java`: declare, initialize, print, traverse, reverse, sort, copy, and handle 2D arrays.
- `BinaryTreeBasicsDemo.java`: create nodes, insert into a BST, traverse using inorder, preorder, postorder, and level-order, and answer common basic tree questions.

## Suggested practice order
1. Arrays
2. LinkedList
3. HashMap
4. BinaryTree

## Interview reminder
- For printing a map key and value together, `entrySet()` is the most common answer.
- For printing arrays, `Arrays.toString(...)` is the most common answer.
- For trees, know the difference between inorder, preorder, postorder, and level-order.

## One-line memory anchors
- Array: fixed-size indexed collection.
- LinkedList: node-based list with easy add/remove at ends.
- HashMap: key-value store with fast average lookup.
- Binary tree: node with left and right child; traversal order matters.
